# Impeccable Design Context — whatthediff

## Design Context

### Users
Developers, SREs, DevOps and security engineers evaluating a CLI tool. They
arrive from GitHub, Hacker News, or a search like "config drift tool" /
"semantic diff many files". Context: desk, daylight, quick evaluation —
they decide in ~30 seconds whether to run the install one-liner.

### Brand Personality
Calm, exact, evidence-first. Three words: **precise, quiet, verifiable**.
The product's whole thesis is "claims backed by proof" — the site must feel
the same: real numbers, real terminal output, zero hype adjectives.

### Aesthetic Direction
Explicit user direction: styled after **radix-ui.com** — light theme, white
surfaces, Radix Colors (slate neutrals, blue accent, the signature pastel
scale chips), Radix Icons (15×15), restrained single-sans typography with
weight-driven hierarchy, generous whitespace, hairline borders.
Anti-references: dark "hacker" landing pages, neon gradients, glassmorphism,
icon-card grids.

### Design Principles
1. Proof over adjectives — every claim on the page is a measured number or
   real command output.
2. The terminal output IS the hero artwork — real corpus, real drift table.
3. Radix's visual language: light, hairline-bordered, weight-not-color
   hierarchy, one blue accent used sparingly; bucket colors (green/blue/
   amber/red-ish) only where they carry meaning (consensus buckets).
4. One action: the copy-able install one-liner. Everything funnels to it.
5. Fast + static: no framework, no build step, self-contained assets,
   SEO/LLM-crawler complete (JSON-LD, OG, sitemap, llms.txt).
